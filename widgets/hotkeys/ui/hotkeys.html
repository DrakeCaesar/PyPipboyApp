^ Basics

This widget allows you to use hotkeys to trigger various in game actions. A small set of predefined hotkeys are included as defaults, which you can customise:

You can delete an existing hotkey by selecting it in the list, and then clicking the 'Delete Hotkey' button.

To create a new hotkey, fill in the fields below the table, and then click 'Add New Hotkey'.

To save your changes so that they are remembered when you next start PyPipBoyApp, click the 'Save Hotkeys' button.

If you've saved changes to the hotkey list and want to revert to the defaults, delete each of the hotkeys, click Save Hotkeys, and restart PyPipBoyApp.

If you assign more than one action to the same hotkey, all of those actions will be triggered (in the order they were defined).

^ Troubleshooting and Limitations

There are number of limitations to the configurations in which the hotkeys will work, all of the following constraints must be met:

  • PyPipBoyApp must be:
    • running on Windows.
    • running on the same machine as Fallout 4.
    • running at the same or higher privilege level as Fallout 4 (ie: if Fallout 4 is run 'as admin' then PyPipBoyApp must also be run 'as admin').
  • Hotkeys are only processed when Fallout 4 has focus\is the active window.
  • Hotkeys are only processed during open gameplay (not in the menus, not looking at the pipboy, not in dialogue or lockpicking, etc). Generally, if you can fire your weapon then the hotkeys should work, if you can't then they shouldn't.
  • Most hotkeys will only work correctly if your in game language is set to English.

If you have checked that all of the above constraints are met but are still having problems post a message on the Nexus page including the following information:

  • Whether any of the non-hotkey functionality of PyPipBoyApp is working (ie: the world map, player status, etc)
  • Whether any of the default hotkeys work.
  • Whether the 'Test Hotkey Hook' action causes the "Success!!" message to show in the Widget.
  • Which version of Windows (7, 8, 10, etc) including bitness (32 or 64) you are using.

^ Available Actions

^ Cycle Equipped Grenades
Cycles your currently equipped grenade through each of the different grenades and mines in you are carrying (in alphabetical order).
Parameters: None
^ Unequip\Equip Current Grenade
Unequip the currently equipped grenade, or equip the last equipped grenade. If no grenade has been equipped, equips the first one available.
Parameters: None
^ Save all currently equipped apparel to slot

Saves all of you currently equipped items of apparel as a set to a number slot, so they can quickly equipped later.

Items sets are persisted across restarts, and any existing set is automatically overwritten when you save to that slot.

Parameters:
  1. The slot number in which to save the set of apparel. This must be a number from 1 to 99.
^ Equip apparel from a saved slot

Equips all the items of apparel that were previously saved to that numbered slot. If you are no longer carrying any of the saved items (because they've been dropped or renamed for example) they will be ignored.

Parameters:
  1. The slot number from which to load the set of apparel. This must be a number from 1 to 99.
^ Unequip all items of apparel
Unequip all currently equipped items of apparel, useful if some of your saved sets of apparel do not include all types of item.
Parameters: None
^ Toggle Radio On\Off
Toggles your pipBoy radio on and off, preserving current station.
Parameters: None
^ Tune to next radio station
Tunes your pipBoy radio to the next active and in range signal.
Parameters: None
^ Use Stimpak
Uses a Stimpak(surprise!), if you have any.
Parameters: None
^ Use Radaway
Uses a RadAway, if you have any.
Parameters: None
^ Use Jet
Take a hit of Jet, if you have any.
Parameters: None
^ Use Named Item

Use any item identified by name and the Inventory section in which to find it. The exact meaning of use depends on item: drugs\food will be consumed, weapons\armour will be equipped (or unequipped if already equipped).

Binding multiple instances of this command to a single hotkey can be used to switch between sets of armour, or take multiple aid items at once.

Note: as this works by matching the name of the item, it will stop working if an item is renamed (ie: when armour\weapons are modified).

Parameters:
  1. The inventory section to search for the item named in the second parameter. You can use the DataBrowser Widget to locate your item and determine the correct section, but generally: 29 for apparel; 43 for weapons; 48 for aid.
  2. The fullname of the item you want to use, without quotes. This is a case insensitive string comparison, so (other than case) the name must match exactly.
^ Toggle Hotkeys On/Off
Disable/Reenable all hotkeys, potentially useful if you've mapped keys already used by the game in certain situations. By default all configured hotkeys are enabled.
Parameters: None
^ Test Hotkey Hook
Test function to assist with troubleshooting. Causes the text: "Success!!" to appear above the table in the widget if the key press is successfully detected by PyPipBoyApp.
Parameters: None
